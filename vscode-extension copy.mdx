## Overview

The Lyceum Cloud VS Code extension enables developers to execute Python code and Jupyter notebooks directly on cloud infrastructure from within their familiar development environment.

## Key Features

### Code Execution
- **Execute Python Files**: Run entire Python scripts on cloud instances with a single command
- **Jupyter Notebook Support**: Execute individual notebook cells on remote infrastructure  
- **Real-time Output**: View execution results, logs, and errors in real-time
- **Multiple Machine Types**: Choose from various CPU and GPU configurations

### File Management
- **Upload to Cloud**: Upload local files and directories to cloud storage
- **Download Results**: Retrieve output files, datasets, and generated content
- **Cloud File Browser**: View and manage files stored in cloud storage
- **S3 Integration**: Direct integration with S3-compatible storage systems

### Execution Management
- **Active Executions**: Monitor currently running workloads
- **Execution History**: View past executions with detailed logs and metadata
- **Abort Operations**: Stop running executions when needed
- **Resource Monitoring**: Track compute resource usage and costs

## Installation & Setup

### Install Extension
1. Open VS Code
2. Go to Extensions (Ctrl+Shift+X)
3. Search for "Lyceum Cloud"
4. Click Install on the official Lyceum Cloud extension

### Configure Authentication
1. Get your API token from the [Dashboard](https://dashboard.lyceum.technology)
2. Open Command Palette (Ctrl+Shift+P)
3. Type "Lyceum: Configure API Token"
4. Paste your API token

### Set API URL (Optional)
Configure the Lyceum Cloud API endpoint if using a custom installation:
1. Open Command Palette (Ctrl+Shift+P) 
2. Type "Lyceum: Configure API URL"
3. Enter your API endpoint (default: https://api.lyceum.technology/api)

## Usage

### Executing Code

#### Python Files
- **Toolbar Button**: Open a Python file and click the cloud icon in the editor toolbar
- **Keyboard Shortcut**: Use `Ctrl+Shift+E` to execute the current file
- **Command Palette**: Use `Ctrl+Shift+P` → "Execute on Cloud"

#### Jupyter Notebooks
- **Cell Execution**: In Jupyter notebooks, click "Execute on Cloud" button on any code cell
- **Real-time Results**: View output and logs as they stream from the cloud

### File Operations

#### Upload Files
- **Right-click Upload**: Right-click files/folders in Explorer → "Upload to Lyceum Cloud"
- **Keyboard Shortcut**: Use `Ctrl+Shift+U` to upload selected files
- **Drag & Drop**: Drag files to the "Lyceum Cloud Files" panel

#### Manage Cloud Files
- **File Browser**: View uploaded files in the "Lyceum Cloud Files" panel in the Explorer
- **Download Files**: Click download icon next to any cloud file
- **Delete Files**: Click trash icon to remove files from cloud storage

### Execution Management

#### Monitor Active Runs
- **Management Panel**: Use `Ctrl+Shift+M` to open execution management panel
- **Real-time Updates**: See live status updates and log streaming
- **Resource Usage**: Monitor CPU, memory, and cost information

#### Access History
- **Execution History**: View all past executions with searchable history
- **Detailed Logs**: Access complete execution logs and metadata
- **Cost Analysis**: Track spending and resource usage over time

| Command | Shortcut | Description |

| Execute on Cloud | `Ctrl+Shift+E` | Run current Python file or notebook cell |

| Manage Executions | `Ctrl+Shift+M` | Open execution management panel |
| Configure API Token | - | Set authentication token |
| Configure API URL | - | Set custom API endpoint |

## Configuration Options

### Extension Settings
Access settings through VS Code Preferences → Extensions → Lyceum Cloud:

- **API URL**: Base URL for Lyceum Cloud API
- **JWT Token**: Authentication token for API access  
- **Keepalive Interval**: SSE connection maintenance interval (5-300 seconds)
- **Default Machine Type**: Preferred execution environment
- **Auto-upload Dependencies**: Automatically upload requirements.txt files

### Workspace Configuration
Configure per-workspace settings in `.vscode/settings.json`:

```json
{
  "lyceum.apiUrl": "https://api.lyceum.technology/api",
  "lyceum.defaultMachineType": "cpu-small",
  "lyceum.autoUploadDeps": true
}
```

## Best Practices

### Code Preparation
- **Dependencies**: Clearly specify required packages in requirements.txt
- **File Paths**: Use relative paths for better portability
- **Output Handling**: Save important results to files for download
- **Resource Awareness**: Consider memory and compute requirements

### File Management
- **Organization**: Use clear folder structures in cloud storage
- **Cleanup**: Regularly remove unnecessary files to optimize storage costs
- **Security**: Be mindful of sensitive data in cloud storage

### Performance Tips
- **Batch Operations**: Upload multiple files at once when possible
- **Stream Large Outputs**: Use print statements for progress monitoring
- **Resource Selection**: Choose appropriate machine types for your workload

## Troubleshooting

### Authentication Issues
- **Invalid Token**: Verify API token is correctly configured
- **Expired Session**: Re-authenticate if login session has expired
- **Network Issues**: Check internet connectivity and firewall settings

### Execution Problems
- **Dependencies**: Ensure all required packages are specified
- **Resource Limits**: Verify selected machine type has sufficient resources
- **Code Errors**: Review execution logs for syntax or runtime errors

### File Operation Issues
- **Upload Failures**: Check file size limits and network connectivity
- **Permission Errors**: Verify S3 credentials and bucket permissions
- **Download Issues**: Ensure sufficient local disk space
